## 2. NEW — Starting Frame Realism Analysis (P0)

### Problem
Khalil's review of the Apex Hunter clips confirms that starting frames (generated by Gemini) look like **illustrations or concept art**, not photorealistic film stills. Since Higgsfield's video models animate FROM the provided starting frame, the non-realistic style of the input image propagates through the entire clip. This is the single biggest quality gap between the auto-pipeline output and the blockbuster-quality target.

### Root Cause Analysis

**Current prompt** (from `src/lib/generate-image.ts`, line 531-551):
```
Generate a PHOTOREALISTIC FIRST FRAME for a film shoot — not a storyboard 
illustration, not a concept sketch.
...
Style: photorealistic cinematic frame, natural film grain, shallow depth of 
field where appropriate, production-quality color grading, NOT illustrated 
or animated.
```

The prompt text is correct — it explicitly asks for photorealistic and explicitly rejects illustration. The problem is likely multi-factor:

1. **Model tier**: Using `gemini-3.1-flash-image-preview` (Flash) — the Flash tier may not achieve the same photorealism as Pro or larger models. Investigate whether `gemini-2.5-pro-preview` or `gemini-3.1-pro-image-preview` produces more realistic output.

2. **No realism validation**: After generating the first frame, there is NO automated check to verify the output is actually photorealistic. The auto-selection step (`auto-select.ts`) uses Claude Haiku vision scoring, but it scores against the textual brief — it does not specifically evaluate photorealism vs. illustration style.

3. **Fantasy genre bias**: For Apex Hunter specifically, the fantasy/dark-fantasy setting (dragons, medieval armor, magic) may push Gemini toward concept-art/illustration style since its training data likely contains far more fantasy illustrations than photorealistic fantasy content.

4. **Missing negative instruction specificity**: The prompt says "NOT illustrated" but doesn't specify what to avoid: flat/clean color fills, perfect symmetry, smooth skin with no pores, saturated color palettes, linework edges, etc. More detailed negative guidance may help.

### Recommended Fix — Realism Gate Step

Add a new pipeline step between `first_frames` and `video_clips`:

```
Step 11.5: realism_gate
For each generated first frame:
  1. Send frame to Gemini 2.5 Pro vision with structured prompt:
     "Score this image 1-10 for PHOTOREALISM. A 10 is indistinguishable from 
      a DSLR photograph on a film set. A 1 is a cartoon or illustration. 
      Evaluate: skin texture/pores, lighting physics, material properties, 
      edge definition, color palette naturalism, depth of field. 
      Return JSON: { score: number, issues: string[], style: 'photorealistic' | 'illustration' | 'mixed' }"
  2. If score < 7 OR style !== 'photorealistic':
     a. Regenerate with enhanced prompt adding specific anti-illustration instructions
     b. Include the FAILED frame as a negative reference: "Do NOT produce an image like this — it looks illustrated"
     c. Up to 3 retries before marking as needs_manual_review
  3. Log scores for QA dashboard
```

### Prompt Enhancement (Apply Regardless of Gate)
Add to the first frame prompt:
```
CRITICAL REALISM REQUIREMENTS:
- Skin must show pores, subsurface scattering, and micro-imperfections
- Lighting must follow physically correct behavior (soft shadows, specular highlights on metal, caustics)
- Materials must have tactile texture (fabric weave, leather grain, steel reflection)
- Color palette must be desaturated and naturalistic — NO oversaturated fantasy colors
- NO clean vector-like edges, NO flat color fills, NO perfect symmetry
- This must look like it was captured by an Arri Alexa 65 on a practical film set
```

### Supercomputer "Realism Pass" Process
During the manual Supercomputer phase for Apex Hunter, a realism verification process was performed where each character, prop, and scene reference was checked for photorealism before being used in generation. The specific process:
1. Generate reference image
2. Evaluate whether it looks like a real photograph vs. concept art
3. If illustration-like: regenerate with stronger realism directives, or use a different model
4. Only proceed once the reference passes visual inspection

This manual process needs to be codified as the automated `realism_gate` step described above.

## 4. NEW — Gemini Video Analysis Capabilities (Confirmed)

### Research Summary
Khalil asked whether Gemini can analyze video clips and score them for quality/realism. **Answer: YES — this is fully supported and production-ready.**

### Gemini 2.5 Pro Video Understanding — Key Capabilities
- **Native video input**: Process video files directly via the API (upload, inline base64, or YouTube URL)
- **Multiple videos per request**: Up to 10 videos in a single API call (Gemini 2.5+)
- **Audio + visual analysis**: Processes both audio and visual streams simultaneously
- **Timestamp-based queries**: Can analyze specific moments at MM:SS precision
- **Custom FPS**: Default 1 FPS sampling, adjustable up to 5+ FPS for fast-action content
- **Long-form support**: Up to 1 hour at default resolution, ~6 hours at low resolution with 2M token context
- **Small file inline**: Clips under 20MB can be passed directly as base64 — our clips are 4-6 seconds, well under this limit

### Video Quality Scoring Pipeline (Recommended Implementation)

```typescript
// Step 12.5: video_quality_gate (after video_clips, before assemble)
async function scoreVideoClip(clipBase64: string): Promise<{
  realism_score: number;        // 1-10 photorealism
  motion_quality: number;       // 1-10 smooth motion
  character_consistency: number; // 1-10 identity maintained
  issues: string[];
  overall: number;
}> {
  const ai = new GoogleGenAI({ apiKey });
  const response = await ai.models.generateContent({
    model: "gemini-2.5-pro-preview-06-05",
    contents: [{
      parts: [
        { inlineData: { mimeType: "video/mp4", data: clipBase64 } },
        { text: `Analyze this AI-generated video clip for a blockbuster film production.
          Score each dimension 1-10:
          1. REALISM: Does this look like footage from a real film set (10) or an animated/illustrated sequence (1)?
          2. MOTION QUALITY: Is the motion smooth and natural (10) or jerky/warped (1)?
          3. CHARACTER CONSISTENCY: Do characters maintain their identity throughout (10) or morph/drift (1)?
          List specific issues found.
          Return JSON: { realism_score, motion_quality, character_consistency, issues[], overall }` }
      ]
    }],
    config: { responseModalities: [Modality.TEXT] }
  });
  return JSON.parse(response.text);
}
```

### Higgsfield Video Analysis (Alternative/Complementary)
Higgsfield also has a built-in `video_analysis_create` MCP tool that performs scene-by-scene analysis. This returns after 3-5 minutes and could be used for:
- Scene segmentation and labeling
- Detecting major transitions
- Quality flags

**Recommendation**: Use **both** — Gemini for realism/quality scoring (customizable prompts), Higgsfield for scene-level structural analysis.

## 6. UPDATED — Rights Verification Gate (P1)

### Problem
Some Higgsfield video generations require the user to click a "rights verification" button on the platform before the video is released. The auto-pipeline's `pollHiggsfieldJob()` function recognizes these statuses: `completed`, `failed`, `error`, `nsfw`, `queued`, `in_progress` — but does NOT handle `rights_verification`.

### Impact
When a clip enters rights verification, the polling loop treats it as `in_progress` until it times out (~200 seconds), then returns `pending_external`. The clip is never retrieved even though it was successfully generated.

### Fix
Add `rights_verification` to the status handler in `pollHiggsfieldJob()`:
```typescript
case "rights_verification":
  // Mark as needs-manual-action and move on
  return { status: "rights_verification", url: null };
```
Surface this in the QA step so Khalil knows which clips need manual verification clicks.

## 9. "Ash" Name-Noun Collision (P2)

`src/lib/prompt-engine.ts` → `applyElementPlaceholders()` uses case-insensitive regex (`"gi"` flag) to swap character names for `<<<element_id>>>` placeholders. The name "Ash" matches common English words like "ash" in "blood and ash", "ash-covered ground", etc.

**Fix**: Use word-boundary anchors: `new RegExp(`\\b${name}\\b`, "gi")` and/or exclude names shorter than 4 characters from case-insensitive matching.

## 10. Play All Button Fix (P2)

`src/app/projects/[id]/video/watch/page.tsx`:
- **Bug**: `key={current.clip_id}` on the `<video>` element causes React to unmount/remount on clip change
- **Effect**: `.play()` fails because the new element doesn't inherit the user gesture context from the previous element's ended event
- **Fix**: Remove the `key` prop, use a persistent `<video>` element that swaps `src` and calls `.load()` then `.play()`

## 12. Gemini Video Analysis — Implementation Guide

### Why This Matters
The pipeline currently has NO automated quality check on generated video clips. The QA step scores based on metadata (duration, resolution, prompt match) but does not analyze the actual video content. Gemini 2.5 Pro's video understanding capability enables a true visual quality gate.

### API Details (from Google AI documentation, verified June 2026)

**Input methods for short clips (our use case):**
```javascript
// Inline base64 — best for clips under 20MB (ours are ~2-5MB)
const response = await ai.models.generateContent({
  model: "gemini-2.5-pro-preview-06-05",
  contents: [{
    parts: [
      { inlineData: { mimeType: "video/mp4", data: clipBase64 } },
      { text: "Analyze this video clip..." }
    ]
  }]
});
```

**Multi-clip batch analysis (Gemini 2.5+ supports up to 10 videos per request):**
```javascript
// Batch — analyze up to 10 clips in one call
const parts = clips.flatMap((clip, i) => [
  { inlineData: { mimeType: "video/mp4", data: clip.base64 } },
  { text: `Clip ${i + 1} (Panel ${clip.panelNumber}):` }
]);
parts.push({ text: "Score each clip for realism, motion quality, and character consistency..." });
```

**Token cost per clip:**
- ~300 tokens/second at default resolution (1 FPS + audio)
- A 5-second clip ≈ 1,500 tokens input
- 13 clips × 1,500 = ~19,500 tokens — well within single-request limits

### Recommended Scoring Rubric
```
REALISM (1-10):
  10: Indistinguishable from DSLR footage on a practical film set
   8: Clearly generated but photorealistic — minor tells (lighting, texture)
   5: Mixed — some photorealistic elements, some illustrated
   3: Predominantly illustration/concept art style
   1: Cartoon, cel-shaded, or fully animated

MOTION QUALITY (1-10):
  10: Smooth, physically correct motion, no artifacts
   8: Minor warping or acceleration glitches
   5: Noticeable jitter, limb distortion, or unnatural movement
   3: Significant morphing, identity drift mid-clip
   1: Unwatchable — extreme warping or static

CHARACTER CONSISTENCY (1-10):
  10: Characters perfectly match reference identity throughout
   8: Minor drift in details (hair, clothing)
   5: Same person but wardrobe/features shift noticeably
   3: Identity unclear or changes mid-clip
   1: Completely different person
```
